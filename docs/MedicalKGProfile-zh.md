# 中文临床指南医学知识图谱 Profile

本文说明 `MEDICAL_KG_PROFILE=clinical_guideline_zh` 的行为、适用范围和重建要求。

## 目标

该 profile 面向中文诊疗方案、临床指南、专家共识和公共卫生规范文本。目标不是把原文里的每个词都做成节点，而是抽取对检索和问答稳定有价值的医学概念，并用医学语义组织图谱。

核心原则：

- 只抽取原文明确支持的实体和关系。
- 疾病、病原体、症状、并发症、药物、诊断、预防、人群、指南建议等概念优先保留。
- 剂量、时间窗、实验室数值等一次性值不作为独立节点，优先保留在关系描述中。
- 同义词和简称尽量合并，例如 `流感` 合并到 `流行性感冒`。
- 对指南类文档，优先保留 `指南/方案 -> 推荐/措施/方案 -> 适用对象` 的证据链。

## 启用方式

在 `.env` 中设置：

```env
MEDICAL_KG_PROFILE=clinical_guideline_zh
ENTITY_EXTRACTION_USE_JSON=true
```

`clinical_guideline_zh` 会自动使用 `PROMPT_DIR/entity_type/医学实体类型提示词.yml` 作为实体关系抽取提示词。`PROMPT_DIR` 默认是 `./prompts`；如果你像当前本机配置一样把提示词放在 `./data/prompts`，需要显式设置 `PROMPT_DIR=./data/prompts`。只有在需要自定义提示词时才设置 `ENTITY_TYPE_PROMPT_FILE`，显式设置会覆盖 profile 的默认提示词。

## 工作区重建要求

该 profile 会影响抽取、归一化、层级补全和 medical view 投影。已有 workspace 中已经入库的图谱不会自动重抽。

切换到该 profile 后，建议使用新的 workspace 或清空旧 workspace 后重建：

```env
WORKSPACE=tzp_medical_v1
```

保守的干净重建步骤见 [医学 KG 干净 Workspace 重建指南](MedicalKGCleanWorkspace-zh.md)。删除或备份旧数据时，只处理配置目录下的目标 workspace 数据。

如果同时切换了 embedding 模型，也必须清空对应 workspace 的向量和图谱数据；不同 embedding 模型生成的向量空间不能混用。

## 图谱结构

内置后处理会进行：

- 实体归一化：合并常见简称、同义词和英文缩写。
- 值节点清理：删除剂量、时间、指标等不适合作为实体的节点，并把信息保留到相关描述中。
- 层级补全：为已抽取的子类补充医学父节点和 `属于` 关系。
- 医学视图分组：`/graphs?...&medical_view=true` 返回 `metadata.medical_groups`，用于 WebUI 按医学类别显示邻接关系；需要层级浏览时，叠加 `medical_browse=true`。

当前分组包括：

- `pathogen`: 病原体、病毒型别、亚型、毒株。
- `symptom`: 症状、体征、临床表现及症状组。
- `complication`: 并发症、严重后果、器官损伤。
- `diagnosis`: 检查、检验、检测方法、诊断/分型/重症判定标准。
- `treatment`: 药物、治疗方案、支持治疗和中医药治疗。
- `prevention`: 疫苗、隔离、监测、院感控制等预防措施。
- `population`: 人群、特殊风险对象、高危因素。
- `guideline`: 指南、诊疗方案、推荐意见。
- `other`: 疾病中心节点或暂不适合以上类别的节点。

## 质量检查

重建后建议检查：

- `流行性感冒` 仍是疾病中心节点，通常位于 `other`。
- `A(H1N1)pdm09/A(H3N2)/Victoria系 -> 甲型流感病毒/乙型流感病毒 -> 流感病毒` 出现在病原体结构中；层级边方向为“子类 -> 父类”。
- `全身症状/呼吸道症状/消化道症状` 能承接具体症状。
- `呼吸系统并发症/心脏并发症/神经系统并发症` 能承接具体并发症。
- `奥司他韦/帕拉米韦/玛巴洛沙韦` 等药物不应与剂量值拆成孤立实体。
- `75 mg`、`5天`、`1周` 等值不应成为主要实体节点。

## 质量阈值

验收一个中文临床指南 workspace 时，建议至少满足：

- 值型节点数量为 0。
- 明确同义词重复数量为 0。
- 关键病原体层级存在。
- 关键症状分组存在。
- 关键并发症分组存在。
- `流行性感冒` 直接邻居中不能出现孤立剂量、阈值、页码、表格碎片。

这些阈值用于判断 profile 是否把医学概念、证据描述和值型条件放到了合适的位置。若某个指标未达标，应优先检查提示词、同义词表、层级规则和旧 workspace/cache 是否复用。

## 不适用范围

该 profile 当前不是完整医学结构化抽取器，不适合直接承担：

- 药品说明书全量结构化。
- 病历时间线。
- 真实世界队列数据。
- 医嘱、处方、检验检查结果的逐字段标准化。
- 药物相互作用、禁忌证和剂量调整规则的完整规则引擎。

## 局限

`clinical_guideline_zh` 是通用中文临床指南 profile，但当前内置同义词和层级种子优先覆盖流感诊疗方案场景。扩展到其他疾病指南时，应继续补充对应领域的同义词、父子层级和质量测试。
